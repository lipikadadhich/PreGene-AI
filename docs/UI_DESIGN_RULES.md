# UI DESIGN RULES

## Project

PreGene-AI

AI-powered Healthcare Platform

---

# Design Philosophy

The application should look like a real healthcare platform.

It should communicate:

- Trust
- Scientific credibility
- Professionalism
- Simplicity
- Modern healthcare

It should never resemble:

- Student dashboard
- Admin panel
- SaaS template
- Crypto website
- Gaming UI

---

# Design Keywords

- Clinical
- Premium
- Clean
- Minimal
- Spacious
- Professional
- Calm
- Accessible

---

# Color Palette

Primary

Blue

Secondary

Emerald

Neutral

White

Slate

Gray

Avoid

- Neon colors
- Bright gradients
- Dark mode
- Overly saturated colors

---

# Typography

Large page headings

Comfortable spacing

Readable body text

Clear visual hierarchy

Never overcrowd pages.

---

# Cards

Rounded corners

Soft shadows

Light borders

Generous padding

Consistent spacing

---

# Buttons

Primary

Solid Blue

Secondary

Outlined

Danger

Red only when required

---

# Icons

Use Lucide React icons.

Icons should be simple and healthcare appropriate.

---

# Animations

Use Framer Motion.

Animations should be:

- Smooth
- Short
- Professional

Avoid flashy effects.

---

# Spacing

Use generous white space.

Avoid dense layouts.

Keep sections visually separated.

---

# Forms

Simple

Accessible

Large click targets

Clear labels

Consistent validation

---

# Tables

Minimal borders

Alternating spacing

Readable typography

No unnecessary colors.

---

# Charts

Simple

Minimal

Clinical

Avoid decorative graphs.

---

# Components

Always reuse existing components before creating new ones.

DiseasePredictionPage.tsx is the primary design reference.

---

# Development Rules

One page per implementation session.

Review before merge.

Do not redesign working components without approval.

Maintain consistency across every page.

When a judge opens the application, the first impression should be:

"This looks like a real AI healthcare platform."

"This looks like a college project."
